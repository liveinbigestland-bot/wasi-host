# wasi-host 安装部署说明

## 1. 项目简介

wasi-host 是一个基于 Zig + wasm3 的 WASM 插件宿主程序，支持多插件并行运行、沙箱隔离、宿主机硬件信息采集等功能。

---

## 2. 环境要求

### 2.1 编译环境

| 工具  | 版本           | 说明                            |
| --- | ------------ | ----------------------------- |
| Zig | 0.13.0       | 编译器（自带交叉编译能力）                 |
| 系统  | Linux x86_64 | 本仓库自带 zig-linux-x86_64-0.13.0 |

### 2.2 运行环境

| 架构              | 支持情况   | 说明                              |
| --------------- | ------ | ------------------------------- |
| x86_64 (amd64)  | ✅ 完整支持 | 常见 PC / 服务器                     |
| aarch64 (arm64) | ✅ 完整支持 | 树莓派 3B+ / 4 / 5，以及其他 ARM 64 位设备 |
| armv7l (arm32)  | ✅ 支持   | 部分 NAS / 旧款 ARM 设备（如 hinas）     |

---

## 3. 快速编译

### 3.1 编译所有目标架构

```bash
# 进入项目目录
cd wasi-host

# 编译 x86_64 版本
./zig-linux-x86_64-0.13.0/zig build -Dtarget=x86_64-linux-gnu -Doptimize=ReleaseSmall
cp zig-out/bin/wasi-host zig-out/bin/wasi-host-amd64

# 编译 ARM64 版本
./zig-linux-x86_64-0.13.0/zig build -Dtarget=aarch64-linux-gnu -Doptimize=ReleaseSmall
cp zig-out/bin/wasi-host zig-out/bin/wasi-host-arm64

# 编译 ARM 32 位版本（armv7l）
./zig-linux-x86_64-0.13.0/zig build -Dtarget=arm-linux-gnueabihf -Doptimize=ReleaseSmall
cp zig-out/bin/wasi-host zig-out/bin/wasi-host-arm32
```

编译产物位于 `zig-out/bin/` 目录。

### 3.2 ARM 32 位编译说明

**⚠️ 注意：** ARM 32 位（arm-linux-gnueabihf）编译时，LLD 链接器会输出以下 warning：

```
ld.lld: warning: Linking two modules of different target triples:
'arm-unknown-linux-gnueabihf' vs 'armv6kz-unknown-linux-gnueabihf'
```

这是 Zig 0.13.0 的一个已知行为。二进制文件仍能正常生成（已验证可执行），仅为 LLD 链接器的 target triple 匹配度提示，不影响运行。如果 `zig build` 因此返回非零退出码，可忽略并直接使用产物文件。

如果上述 `zig build` 方式不成功，可以用 `zig build-exe` 直接编译：

```bash
./zig-linux-x86_64-0.13.0/zig build-exe \
    -target arm-linux-gnueabihf \
    -O ReleaseSmall -fstrip -lc \
    -I wasm3/source \
    -cflags -DM3_ENABLE_WASI=1 -Dd_m3HasWASI=1 -DM3_HAS_TAIL_CALL=0 -std=gnu11 -- \
    wasm3/source/m3_core.c \
    wasm3/source/m3_env.c \
    wasm3/source/m3_exec.c \
    wasm3/source/m3_compile.c \
    wasm3/source/m3_parse.c \
    wasm3/source/m3_bind.c \
    wasm3/source/m3_code.c \
    wasm3/source/m3_module.c \
    wasm3/source/m3_function.c \
    wasm3/source/m3_info.c \
    wasm3/source/m3_api_wasi.c \
    wasm3/source/m3_api_libc.c \
    wasm3/source/extensions/m3_extensions.c \
    main.zig \
    --name wasi-host
```

## 4. 编译参数说明

### 4.1 `-Dtarget=<triple>`

指定交叉编译的目标平台。Zig 内置交叉编译能力，无需安装其他工具链。

| 参数值                   | 目标平台                        | 适用设备               |
| --------------------- | --------------------------- | ------------------ |
| `x86_64-linux-gnu`    | x86_64 Linux                | 常见 PC / 服务器        |
| `aarch64-linux-gnu`   | ARM 64 位 Linux              | 树莓派 3B+ / 4 / 5    |
| `arm-linux-gnueabihf` | ARM 32 位 Linux (hard float) | 部分 NAS / 旧款 ARM 设备 |

> 查看目标机器架构：在设备上运行 `uname -m`
> 
> - 输出 `x86_64` → 选 `x86_64-linux-gnu`
> - 输出 `aarch64` → 选 `aarch64-linux-gnu`
> - 输出 `armv7l` → 选 `arm-linux-gnueabihf`

### 4.2 `-Doptimize=<mode>`

优化级别：

| 值              | 说明           |
| -------------- | ------------ |
| `ReleaseSmall` | 最小体积（生产部署推荐） |
| `ReleaseFast`  | 最快运行速度       |
| `ReleaseSafe`  | 带安全检查的发布模式   |
| `Debug`        | 调试模式         |

### 4.3 `-fstrip`

剥离调试符号，减小二进制体积。

### 4.4 C 编译标志（build.zig 中配置）

| 宏定义                   | 说明                                 |
| --------------------- | ---------------------------------- |
| `DM3_ENABLE_WASI=1`   | 启用 wasm3 的 WASI 支持                 |
| `Dd_m3HasWASI=1`      | 显式声明 WASI 支持                       |
| `DM3_HAS_TAIL_CALL=0` | **ARM 32 位专用**：禁用 LLVM musttail 优化 |
| `std=gnu11`           | 使用 GNU C11 标准                      |

## 5. build.zig 特殊说明

### 5.1 与原始版本的区别

本项目的 `build.zig` 相比 wasm3 官方集成示例，有以下改动：

**a) 添加 WASM 插件编译步骤**

```zig
const wasm_target = b.resolveTargetQuery(.{
    .cpu_arch = .wasm32,
    .os_tag = .wasi,
});
```

自动编译 `plugins/` 目录下的 Zig 源文件为 wasm32-wasi 插件，并嵌入到宿主机二进制中。

**b) wasm3 C 源文件列表（使用新版 API）**

```zig
exe.addCSourceFiles(.{
    .root = b.path("wasm3/source"),
    .files = &.{ ... },
    .flags = &.{"-DM3_ENABLE_WASI=1", ...},
});
```

使用 `.root` 指定源文件根目录，`addIncludePath` 指定头文件路径。

**c) ARM 32 位兼容**

```zig
.flags = &.{"-DM3_HAS_TAIL_CALL=0", ...}
```

添加了 `-DM3_HAS_TAIL_CALL=0` 宏定义，禁用 wasm3 的 `__attribute__((musttail))` 优化。这是因为 LLVM 的 ARM 32 位后端不支持 musttail，启用会导致编译错误：

```
LLVM ERROR: failed to perform tail call elimination on a call site marked musttail
```

该标志对其他架构（x86_64、ARM64）无影响，可以全局保留。

## 6. 部署

### 6.1 部署到远程机器

使用项目根目录的 `deploy.bat`（Windows）：

```bash
# 双击 deploy.bat，按提示输入密码
```

或手动部署：

```bash
# 复制到目标机器
scp zig-out/bin/wasi-host-arm32 root@<目标IP>:/root/wasi-host/wasi-host
scp config.json root@<目标IP>:/root/wasi-host/

# 设置执行权限
ssh root@<目标IP> "chmod +x /root/wasi-host/wasi-host"
```

### 6.2 启动

```bash
ssh root@<目标IP>
cd /root/wasi-host
./wasi-host
```

程序会自动读取同目录下的 `config.json` 加载并运行 WASM 插件。

## 7. 配置文件说明 (config.json)

```json
{
  "plugins": [
    {
      "name": "插件名称",
      "embed_path": "插件 WASM 文件路径（编译时嵌入）",
      "mem_kb": 512,         // 分配内存（KB）
      "timeout_ms": 3000,    // 超时时间（毫秒）
      "network": false,      // 是否允许网络访问
      "write": false,        // 是否允许写文件系统
      "allow_host_info": true // 是否允许读取宿主机信息
    }
  ]
}
```

### 字段说明

| 字段                | 类型      | 说明                               |
| ----------------- | ------- | -------------------------------- |
| `name`            | string  | 插件标识名称                           |
| `embed_path`      | string  | 对应 `main.zig` 中 `@embedFile` 的路径 |
| `mem_kb`          | number  | WASM 运行时分配的内存上限（KB）              |
| `timeout_ms`      | number  | 插件执行超时（毫秒）                       |
| `network`         | boolean | WASI 网络访问权限                      |
| `write`           | boolean | WASI 文件系统写入权限                    |
| `allow_host_info` | boolean | 是否向插件暴露 CPU / 内存 / 系统信息          |
